大道至简的设计哲学

简约, 追求显示表达, 追求自然的语言

对并行支持

interface, 非侵入式接口, 实现一个接口不需要显示的声明

    1. 只要某个类型实现了接口要的方法, 那么我们说该类型实现了此接口
       该类型对象可赋值给该接口
    2. 作为1的推论, Go语言任何内置对象都可以赋值给空接口interface{}
    3. 支持接口查询

极度简化但完备的OOP 面向对象编程

    废弃了: 继承, 构造析构函数, 虚函数, 函数重载, 默认参数等, 简化了符号访问权限控制, 将隐藏的this指针改成了显示定义的receiver对象
